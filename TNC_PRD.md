## 1. 기획 배경 및 목적 (Executive Summary)

### 1.1 시장의 문제점 (Problem)
* **공유의 마찰 (Friction):** 팀원들이 업무 중 유용한 아티클을 발견해도, 슬랙에 공유하기 위해 **https://play.google.com/store/apps/details?id=textcopy.jdklls.cptxtapp&hl=ko**이라는 4단계의 번거로운 과정을 거쳐야 함.
* **정보의 휘발:** 귀찮음으로 인해 URL만 투척되어 팀원들이 읽지 않거나, 아예 공유되지 않고 개인의 지식으로만 남는 경우가 빈번함.
* [cite_start]**시장 기회:** 2025년 AI 트렌드는 '재미'가 아닌 **'업무 생산성(ROI)'**과 **'워크플로우 통합'**에 집중되어 있음[cite: 318, 424].

### 1.2 제품의 목표 (Objective)
* **"10초 공유 혁신":** 웹 브라우저를 떠나지 않고(Embedded), **클릭 3번** 만에 AI 요약과 인사이트를 팀 슬랙으로 전송한다.
* **지식 자산화:** AI가 표준화된 포맷으로 정보를 가공하여, 팀 슬랙 채널을 '고품질 뉴스 피드'로 전환한다.

## 2. 핵심 사용자 경험 (User Experience Flow)

우리의 UX 전략은 **'비용 효율성(Cost-Efficiency)'**과 **'사용자 통제권(Control)'**을 동시에 잡는 것이다.

1.  **진입 (Trigger):** 브라우저 우측 상단 확장 프로그램 아이콘 클릭.
2.  **대기 (Ready State):** 즉시 AI를 호출하지 않고, **"이 페이지를 요약할까요?"** 묻는 팝업 노출. (실수 방지 및 무료 쿼터 절약)
3.  **실행 (Action):** 사용자가 **[시작하기]** 버튼 클릭 시 AI 분석 시작.
4.  **검토 및 편집 (Human-in-the-loop):** AI가 작성한 **'3줄 요약'** 또는 **'핵심 키워드'**를 확인하고, 필요시 수정하거나 코멘트를 추가.
5.  **전송 (Delivery):** **[전송]** 버튼 클릭 시, 연동된 슬랙 채널로 깔끔한 카드 뉴스 형태로 배달.

## 3. 상세 기능 명세 (Functional Specification)

### 3.1 스마트 비용 방어 (Cost Control UX)
* **기능:** 아이콘 클릭 시 API를 바로 호출하지 않고 **대기 화면(Ready Screen)**을 먼저 렌더링.
* [cite_start]**의도:** 사용자의 조작 실수로 인한 API 비용 낭비를 막고, 사용자가 요약 실행 시점을 능동적으로 결정하게 함[cite: 442].
* **구성:** 현재 페이지 제목 확인 + `[요약 시작]` 버튼 + "오늘 남은 무료 횟수(N/5)" 표시.

### 3.2 듀얼 요약 모드 (Dual Summary Modes)
사용자는 설정에서 두 가지 요약 방식 중 하나를 선택할 수 있음.
* **Mode A: 3줄 요약 (Default)**
    * 줄글 형태의 Bullet Point 3개로 내용을 요약.
    * *용도:* 맥락 파악이 중요한 뉴스나 칼럼.
* **Mode B: 키워드 요약 (New)**
    * 가장 중요한 핵심 단어 3~5개를 **태그(Chip) 형태**로 추출.
    * *용도:* 빠르게 핵심만 훑어봐야 하는 트렌드 리포트나 기술 문서.

### 3.3 편집 및 제약 조건 (Editing & Constraints)
AI의 결과물은 '초안'일 뿐, 최종 품질은 인간이 관리함. 정보 과부하(TMI)를 막기 위해 글자 수를 제한함.
* **요약/키워드:** AI가 작성한 내용을 사용자가 직접 수정 가능 (Editable).
* **인사이트 (Insight):** AI가 도출한 시사점 (수정 가능, **최대 150자 제한**).
* **유저 코멘트 (Comment):** 팀에게 남기는 한마디 (**최대 300자 제한**).
    * *UI:* 입력창 우측 하단에 글자 수 카운터(`45/150`) 표시.

### 3.4 슬랙 연동 및 확장성 (Integration)
* **전송 방식:** MVP 단계에서는 **Webhook URL** 방식을 사용.
* **메시지 포맷:** 슬랙의 **Block Kit**을 활용하여, 단순 텍스트가 아닌 가독성 높은 '카드 UI'로 전송.
* **확장성:** 추후 Discord, MS Teams 등 타 메신저 확장을 고려한 아키텍처로 설계.

## 4. 비즈니스 모델 (Freemium Strategy)

무료 사용자의 진입 장벽은 낮추되, 기업/헤비 유저에게는 확실한 **'편의성'**을 판매함.

| 구분 | Free Plan (무료) | Pro Plan (유료) *[추후 구현]* |
| :-- | :-- | :-- |
| **연동 방식** | **Webhook URL 직접 입력**<br>(슬랙 관리자 페이지에서 생성 필요) | **OAuth 로그인 연동**<br>(버튼 클릭 한 번으로 연결) |
| **채널 선택** | **고정된 1개 채널**로만 전송<br>(Webhook 생성 시 지정됨) | **채널 리스트 선택 가능**<br>(드롭다운에서 자유롭게 변경) |
| **요약 횟수** | 1일 5회 제한 | 무제한 |
| **자동 실행** | 매번 [시작하기] 버튼 클릭 | 아이콘 클릭 시 **즉시 요약 (Auto-start)** |

* **UI 전략:** 설정 화면에 'Pro 기능' 버튼(OAuth, 자동 실행 등)을 노출하되, **자물쇠(🔒) 아이콘**으로 잠가두어 구매 욕구를 자극함.

## 5. 성공 지표 (Success Metrics)

* **정량 지표:**
    * **Week 1 Retention:** 설치 후 1주일 뒤에도 사용하는가? (습관 형성)
    * **Completion Rate:** [요약 시작] 버튼 클릭 대비 [슬랙 전송] 완료 비율.
* **정성 효과:**
    * 팀 내 정보 공유 빈도 증가.
    * "누가 공유했는지" 명확해짐에 따라 구성원의 동기 부여 상승.

## 6. 향후 로드맵 (Roadmap)

* **Phase 1 (MVP):** Chrome Extension 출시 + Webhook 연동 + 기본 요약 기능. **(Current)**
* **Phase 2 (Polish):** 키워드 요약 모드 고도화 + 글자 수 제한 UI + 디자인 퀄리티 업.
* **Phase 3 (Biz):** 유료 결제 모듈 붙이기 + OAuth(Pro 기능) 개발.
